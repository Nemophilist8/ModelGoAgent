你是一名熟悉机器学习项目发布方式、开源协议和商业化模式的专家。请根据以下对一个 ML 项目（包括模型、算法、数据集或系统）的描述，判断该项目的发布模式（open_policy）与发布形式（open_type）。

1. 支持的 open_policy 仅包括以下三种：
- "sell"：项目或其衍生物主要以商业化方式发布或使用，例如付费授权、商业 SDK、闭源模型、收费 API、订阅制 SaaS 等。
- "share"：项目或其衍生物以开放共享为目的发布，例如开源代码、开放模型权重或数据，允许他人再分发、复用或二次开发。
- "personal"：项目仅限个人、内部或学术用途，明确限制商业使用、对外分发或再发布。

判断时请重点关注以下因素：
- 是否允许商业用途
- 是否允许对外分发或再发布
- 是否以盈利或收费为主要目的

2. 支持的发布形式仅包含以下三种：

【raw】
- 如果没有特殊说明，默认为raw

【binary】
- 明确提到提供编译后的模型
- 明确提到以动态库/可执行程序形式发布

【saas】
- 明确提到服务仅存在于服务端、用户通过 API 或网页调用

输出要求：
- 只输出 JSON，不要任何解释或额外文本
- 严格按照以下格式输出：

{{
  "open_policy": "sell" | "share" | "personal",
  "open_type": "raw" | "binary" | "saas"
}}

项目描述：
{input}
